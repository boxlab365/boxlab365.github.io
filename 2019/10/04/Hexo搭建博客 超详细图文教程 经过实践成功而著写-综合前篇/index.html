    写在前面

    很多原来使用Hexo的博主们写下了的教程并没有及时更新，导致后来人一头雾水，挫折感十足
    甚至也不乏没有经过实践而搬砖的人士，引流赚黑心钱…..

    前期认识

    搭建个人博客通常有如下选择

  • 技术型：WorkPress
  • 经济型：GitHubPages + Hexo

    GitHubPages + Hexo

  • WorkPress：一般需要独立域名（收费），对MarkDown不友好
  • GitHubPages + Hexo：免费，使用简单，使用者众多..

    Hexo平台和工具介绍

  • Hexo 是一个基于 Nodejs 的静态博客网站生成器，作者是来自台湾的 Tommy Chen； 同时也是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
  • 注：在安装 Hexo 前，必须安装程序：Node.js、Git

    工作大体流程

  1. 准备工作（安装相关软件&命令符检验&注册github账号）
  2. 本地搭建Hexo静态博客
  3. 将文件提交到github
  4. 绑定&解析主域名
  5. 后续美化&配置“主题模板”
  6. 进一步学习（如MarkDown）

Hexo博客官方网站

  • 官方中文网站：https://hexo.io/zh-cn/

    中期准备

    博客搭建和准备环境（以windows平台演示）

  • 注：根据实际需要安装32位或64位版本（有的网站会自动检测判断你的win版本）；软件遵循一路默认安装原则，直接默认Next下一步（如果不正确，则需要回头检查自己的安装过程）

    配置Node.js环境

  • 下载Node.js安装文件:https://nodejs.org/en/download

    配置Git环境

  • 下载地址：https://git-scm.com

    cmd命令符检测

  • 理论上来讲可跳过，只要上面的操作正确即可 (如下图)

    win+r弹出对话框 输入cmd 打开命令符窗口;

    检测Node.js

  • win+R弹出新打开的窗口，并输入：cmd回车打开命令行界面（注意输入法 英文）
  • 输入 node-v 回车查看是否返回版本号之类的（返回即代表安装成功，下同，反之）
  • 输入 npm-v

    检测Git

  • win+R弹出新打开的窗口，并输入：cmd回车打开命令行界面
  • 输入：git –version

    ###上述软件安装还有其它的方法，这里不再赘说###

    GitHub注册账号&配置

  • 如已经拥有账号并创库，请跳过此步
  • Github官网：https://github.com（灵活使用 浏览器翻译功能）
  • 注册的过程中(如用户名注册不上 请更换)一定要在所填写的邮箱后台，确认github的注册确认信，结束注册流程

    Github创建库

  1. 注册登陆成功并创建库
  2. 进入代码库创建页面，在Repository name下填写yourname.github.io，Description (optional)下填写一些简单的描述（不写也没有关系），README那一项初始化也建议勾上（如下图）
  3. 注意格式为：你注册github的英文昵称.github.io
  4. 点击Settings（如下图）
  5. 下拉，找到GitHub Pages设置界面（如下图）
  6. 点击Choose a theme随便选择一个”杰基尔主题”并提交（如下图）

    7.选择后会自动返回，这里可以编辑主题元素&相关

    ###现在，你已经有了一个简单的github博客;验证创建的博客页面是否成功：在浏览器输入网址成功打开即可###

    Hexo本地搭建

  • 为其统一 全部采用win自带的命令行演示&操作

    其它教程操作可能是在文件夹中 鼠标右击启用：git bash here 或按住shif键+鼠标右击选择：在此处打开powershell窗口

  1. 找个合适的位置 建立文件 用于存放博客文件资源（如G盘建立：myblog）
  2. win+R弹出新打开的窗口，并输入：cmd回车打开命令行界面（注意输入法 英文）
  3. 使用cd命令进入文件夹：（1）cd g:（2）g:（3）cd myblog (如下图 )可直接从(2)开始
  4. 执行Hxeo命令：npm install hexo-cli -g 会有几个报错，无视它就行，但必须确保正常安装了Node.js和Git
  5. 如果出错，然后则输入npm install hexo --save继续安装(如下图)
  6. 安装完后输入hexo -v验证是否安装成功 #此步可跳过，前提是确保上述没有问题存在(如下图)
  7. 输入hexo init xxx 并回车 //xxx可替换成你建立博客的文件夹 （如blog 它在上述myblog的目录之下）该操作其实就是初始化Hexo
  8. 输入 npm install 并回车耐心安装必备的组件
  9. 输入hexo g 生成静态网页体验本地博客服务预览效果（注：暂且使用默认主题）(如下图)
  10. 输入hexo s 打开本地服务器预览，预览网址是：http://localhost:4000/ 成功配置并打开之后就可以看到默认landscape的主题博客啦(如下图)
  • 操作后博客下source中的文件（如md&html)会被解析到新建立的public文件中（public用于存放静态博客文件，相当于网站的根目录）
  • ctrl+c可以关闭预览服务（在win命令行 ）
  • 截至 本地预览本地服务器博客完成

    Hexo博客本地预览功能

    总言之，写完博客后，均需转化成html文件，并通过浏览器查看效果。

  • 在进入博客目录后，执行命令（节省时间 可直接hexo s开始）
  1. hexo g
  2. hexo s
  • 浏览器打开本地端口：http://localhost:4000/ 即可查看博客内容
  • 本地提交到github 使用命令：hexo d(下面会提到)

博客搭建配置&提交github

  1. 打开&修改博客根目录下的_config.yml文件（这是博客的配置文件）在这里你可以修改与博客相关的各种信息。//打开方式可为：记事本
  2. 打开之后，翻到最后几行修改相关配置（如下图）
  • 注:设置时的键值之间一定要有空格（不然轻则报错 重则崩溃）；打开修改时 可以使用快捷键 Ctrl+F精确辅助查找&替换修改（为了防止出错 请记得备份；修改完成后请记得保存（ctrl+s ）；记得repo哪一项 加上git
  1. 上述操作完成的基础上，必须使用下面的命令才能为提交github做准备（换句话说：该命令是提交至github的前提）
  • 输入npm install hexo-deployer-git --save 回车安装自动部署发布工具
  • 输入 hexo clean //清除缓存 本地网页正常情况下可以忽略此条命令
  • 输入hexo g // 生成静态网页
  • 输入 hexo s可以在本地预览效果
  • 输入 hexo d //开始部署提交到github

    hexo clean && hexo g && hexo d //此命令可直接代替上述一次执行到位
    对于新手不推荐此 期间会要求登陆github账号&密码（它会自己弹出 如同登陆QQ般）之后使用浏览器打开github的个人网址，查看网页是否有所变化

配置SSH

  • 为避免重复的提交内容而多次登陆输入，我们仍旧需要在本地创建ssh key
  1. 新建一个命令符窗口 并输入：ssh-keygen -t rsa -C "你注册github的账户邮箱地址"输入后 回车敲击几下即可(如下图)
  • 正确上述操作后，会在User文件夹对应的用户下创建.ssh文件夹，其中有一个id_rsa.pub文件（这类文件一般都是隐藏属性的，需要你手动更改）(如下图)
  1. 采用记事本的方式打开后，复制全部内容key（ctrl+a | ctrl+c）
  2. 登陆github主页，点击Settings，之后选择左边选择SSH and GPG Keys选项(如下图)
  3. 配置github的ssh(其中的title随便填，下面的文本框内容 粘贴在你电脑上生成的key内容 点击添加之后即可)(如下图)
  4. 验证是否绑定本地成功，在命令行中验证，输入指令：ssh -T git@github.com(如下图)
  • 如第一次执行该指令，则会提示是否continue继续，如果我们输入yes就会看到成功信息

    本地配置SSH结束

后期准备

通过GitHub为其Hexo博客绑定顶级域名

  • 域名的购买与其绑定解析

    默认的域名还是:你的账户名.github.io，对于一些朋友而言，可能需要绑定顶级专属域名（顶级域名是需要购买的）在国内平台购买并使用的话，要进行备案流程（非主流域名可能也不支持备案），如果不想备案的话，就得使用国外域名注册商提供的服务；自己权衡价格并在自己能接受的范围内即可 //Github提供的个人网址，百度不收录，但谷歌支持

  • 免费顶级域名申请地址：https://www.freenom.com
  • 推荐（国外）域名申请地址：https://www.namesilo.com
  • 第三方域名解析地址：https://www.dnspod.cn

    配置域名解析

  1. 登录到注册商的控制台，并找到域名列表页面，并对想使用的域名，进行解析
  • 如使用第三方进行解析，则必须填写域名解析商的NS地址，并清除原NS地址（均在域名注册商哪里完成）
  1. 分别添加两个记录，一个主机记录为@、 一个为www;记录类型是CNAME。
  • 记录值都填同一个，即填你的博客主页对应的ip或github的地址(如下图)

    Github域名配置

  1. 打开你的github博客项目，点击settings，拉到下面Custom domain处，填写顶级域名并save保存(如下图)
  2. ping域名（用来判断是否解析成功和查看IP地址等）
  • win+r快捷键 弹出框中输入：cmd 之后在命令行中：ping 主页地址 （如：ping baidu.com）注意空格（如下图）
  1. 打开浏览器在地址栏输入你的个性化域名，将会直接进入你自己搭建的网站
  • 如果报错404 或不能访问等 则需手动创建文件CNAME

    在本地的博客目录下source文件夹中，新建一个CNAME文件 （注：没有后缀名。新建记事本 之后写入内容 保存&重命名去掉后缀即可） 内容为yourdomin.xxx（如:baidu.com） (如下图)

  • 最后分别运行hexo g hexo d上传到github（或：hexo d -g）重新上传后 在预览效果
  • github个人网址绑定顶级域名支持自定义404界面
  • 在这一系列的操作中，包括修改NS服务器、设置解析等等，都需要一定的时间。域名解析短则几分钟，长则24小时，最长不会超过72小时。如超过，请检查自己的配置过程。

    注(1)新建内容的CNAME文件，可手动在后台中通过 upload files按钮上传到Github上博客的目录（此操作完成后，无需在GitHub的Custom domain处填写域名）
    注(2)自定义404界面 可按照自己要求配置或启用Hexo提供的
    注(3)使用Markdown语法写作 必须注意空格问题

结语

  • 当你完成了自己博客之后，相信你的心情跟我刚做完的心情是一样的，即便很累，但是当自己的博客成型之后，自己还是有一个成就感的.那么完成后，以后的路还很长，真正想要自己博客能够积累人气&美观，不但要有好的内容，还要有一定的技术知识；所以认真对待吧，相信你会受益于此的。
